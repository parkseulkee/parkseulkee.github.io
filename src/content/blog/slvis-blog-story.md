## 들어가며

출산 휴가 중이다. 겨울이(2026년 2월생)를 돌보면서 하루하루가 정신없이 지나간다.

그런데 AI 기술에 관심이 많은 개발자로서, 이 시간을 그냥 보내기엔 아쉬웠다. 평소에 해보고 싶었던 것들이 있었다.

- 나만의 **AI 개인 비서** 만들기
- 오래된 **개인 블로그 리뉴얼**하기

문제는 **육아 중이라 노트북을 펼 시간이 거의 없다**는 것.
수유하고, 재우고, 안아주고... 양손이 자유로운 시간은 하루에 몇 시간 되지 않는다.

그래서 찾은 방법이 **모바일로 AI와 대화하면서 개발하기**였다.

---

## Happy — 모바일에서 Claude Code를 쓰다

![Happy 앱 — Claude Code를 모바일에서 사용할 수 있는 오픈소스 클라이언트](/images/blog/happy-app.png)

[Happy](https://happy.engineering)는 Claude Code를 모바일 앱에서 사용할 수 있게 해주는 오픈소스 프로젝트다.

노트북을 펼 수 없는 상황에서 이 앱은 정말 게임 체인저였다. 수유하면서, 겨울이가 잠든 틈에, 소파에 누워서 — 핸드폰으로 Claude와 대화하면서 코드를 작성하고, 파일을 수정하고, 빌드하고, 배포까지 할 수 있었다.


실제로 이 블로그의 거의 모든 작업을 Happy 앱을 통해 진행했다. 처음엔 "이게 되겠어?" 싶었는데, 막상 해보니 놀라울 정도로 잘 됐다.

## 슬비스(SLVIS) — AI 개인 비서 프로젝트

### 왜 만들었나

네이버에서 데이터 엔지니어로 일하면서 AI/LLM 관련 업무를 많이 해왔다. LLMOps 도구도 만들고, 관련 서적도 출판했다. 그러다 보니 자연스럽게 "나만의 AI 비서를 만들어보자"는 생각이 들었다.

아이언맨의 자비스처럼, 내 일상을 체계적으로 관리해주는 비서. 그게 **슬비스**다. SLVIS — **S**mart **L**ogical **V**irtual **I**ntelligence **S**ystem.

### 어떻게 동작하나

슬비스는 **Obsidian vault를 두뇌로 활용**하는 AI 개인 비서다.

- 📅 **일정 관리** — Google 캘린더와 연동하여 일일 브리핑
- 📰 **뉴스 큐레이션** — 긱뉴스 기반 AI/기술 뉴스 + 경제/정치 뉴스를 매일 정리
- 📝 **지식 관리** — Obsidian에 체계적으로 기록하고 연결
- 🌐 **블로그 운영** — 이 블로그도 슬비스가 관리
- 🏠 **일상 어시스턴트** — 일상 전반의 기록과 관리

### 슬비스 아키텍처

![슬비스 아키텍처 다이어그램](/images/blog/slvis-architecture.svg)

별도의 서버가 없다. **Claude Code가 중심**에서 추론하고, 필요한 작업을 CLI 도구들에 위임하는 구조다.

| 도구 | 역할 |
|------|------|
| **Obsidian CLI** | Vault 노트 읽기·쓰기·검색. Vault 자체가 슬비스의 장기 기억이다. |
| **gcalcli** | Google 캘린더 일정 조회·추가. 자연어 명령을 캘린더 API로 연결. |
| **GitHub CLI** | 블로그 commit·push, PR·이슈 관리. |
| **Chrome MCP** | 웹 브라우저 직접 제어. 아티클 수집, 블로그 포스트 작성 시 웹 탐색이 필요할 때 사용. |

CLI 도구를 우선적으로 쓰는 이유는 **토큰 효율** 때문이다. Chrome MCP는 스냅샷을 통째로 읽어야 해서 토큰 소모가 크다. 그래서 웹 접근이 꼭 필요한 작업에만 한정해서 쓴다.

`CLAUDE.md` 파일이 슬비스의 행동 규칙서 역할을 한다. 어떤 도구를 언제 써야 하는지, 어떤 형식으로 기록해야 하는지 등이 담겨 있고, 매 대화마다 이 파일을 기반으로 일관된 동작을 유지한다.

## 블로그 리뉴얼 — Jekyll에서 Astro로

### 기존 블로그의 문제

원래 [parkseulkee.github.io](https://parkseulkee.github.io)는 Jekyll + YAT 테마로 운영하고 있었다. 2022년부터 회고, 기술 글 등 11개의 포스트가 있었는데, 디자인이 마음에 들지 않았고 포트폴리오로 활용하기엔 부족했다.

### 컨셉 — 슬비스가 운영하는 블로그

블로그를 단순히 리뉴얼하는 게 아니라, **슬비스가 슬기님을 소개하는** 컨셉으로 만들었다. 블로그에 들어가면 슬비스 로고가 뜨고, 슬비스가 말하는 것처럼 소개가 나온다.

구조는 [antfu.me](https://antfu.me)를 참고했다. 디자인은 처음엔 [kentcdodds.com](https://kentcdodds.com)의 밝고 통통 튀는 느낌을 생각했는데, 막상 만들다 보니 방향이 달라졌다.

자비스 컨셉을 살리려면 어두운 배경에 사이버틱한 분위기가 맞았다. [Arwes](https://arwes.dev)라는 SF UI 프레임워크에서 영감을 받아, cyan glow, HUD 프레임, 스캔라인 같은 요소들을 직접 구현했다. "AI 비서가 운영하는 블로그"라는 컨셉에 이 디자인이 훨씬 잘 어울렸다.

### 기술 스택

- **Astro** — 정적 사이트 생성기. 빠르고 가볍다.
- **Tailwind CSS** — 유틸리티 퍼스트 CSS. 빠른 스타일링.
- **MDX** — 마크다운 기반 콘텐츠 관리
- **GitHub Pages** — 무료 호스팅 + GitHub Actions 자동 배포

Jekyll에서 Astro로 전환하면서 기존 11개 포스트를 모두 마이그레이션했다. frontmatter 형식 변환, 코드블럭 언어 수정 등 자잘한 작업이 있었지만, Claude가 알아서 처리해줬다.

## 실제 작업 과정

Happy 앱으로 Claude와 대화하면서 진행한 실제 흐름은 이랬다.

### 1단계: 프로젝트 세팅
> "개인 블로그를 Astro로 리뉴얼하고 싶어. 기존 Jekyll 포스트는 유지하고."

Claude가 기존 레포를 클론하고, Astro 프로젝트를 세팅하고, 포스트를 마이그레이션했다.

### 2단계: 디자인 & 컨셉
> "antfu.me 구조 + kentcdodds.com 디자인으로. 슬비스가 운영하는 컨셉."

레이아웃, 컴포넌트, 스타일을 만들고 로컬에서 프리뷰까지 띄워서 스크린샷으로 보여줬다.

### 3단계: 피드백 & 반복
> "리스트가 안 보여" / "행간이 좁아" / "이 문구 수정해" / "LinkedIn 추가해"

핸드폰으로 사이트를 보면서 피드백하면, Claude가 바로 수정하고 빌드해서 배포까지.

### 4단계: 포트폴리오 통합
> "노션 포트폴리오 참고해서 About 업데이트해. 사내 프로젝트는 상세 페이지로."

노션 페이지를 열어서 내용을 가져오고, 프로젝트 상세 페이지 6개를 한 번에 만들었다.

### 5단계: 배포
> "커밋하고 푸시해. Pages로 보이게 해."

GitHub Actions 워크플로우를 만들고, Pages 설정을 변경하고, 배포까지 자동으로.

이 모든 과정이 **모바일 앱에서의 대화**로 이루어졌다.

## 느낀 점

### AI와 함께하는 개발의 미래

솔직히 이 경험은 충격적이었다. 노트북 없이, 핸드폰으로 대화만 하면서 블로그를 처음부터 끝까지 만들 수 있다니.

물론 AI가 모든 걸 완벽하게 해주진 않는다. 마크다운 렌더링이 깨지는 문제, CSS가 적용 안 되는 문제 등 디버깅이 필요한 순간들이 있었다. 하지만 "이거 깨졌어"라고 말하면 원인을 찾고 고쳐주는 과정이 너무 자연스러웠다.

### 육아와 개발의 양립

육아 중에 개발을 한다는 게 사치처럼 느껴질 수 있다. 하지만 Happy 앱 덕분에 **자투리 시간**을 활용할 수 있었다. 겨울이가 잠든 10분, 수유하면서 한 손으로 타이핑하는 시간 — 이 작은 시간들이 모여서 하나의 프로젝트가 완성됐다.

### 앞으로

슬비스는 계속 진화 중이다. 블로그도 계속 채워나갈 예정이고, 슬비스의 기능도 더 확장할 계획이다.

육아도 개발도, 둘 다 포기하지 않는 방법을 찾은 것 같아 뿌듯하다.
